Database Migration Documentation — HSQLDB to PostgreSQL

Artifact ID: 06_database_migration  |  Strategy: Expand-Contract (6 phases)  |  Author: IBM Bob

1. Migration Strategy

The migration follows the Expand-Contract (also known as Parallel-Run) pattern to ensure zero-downtime data migration. The legacy HSQLDB is exported, transformed, and imported into PostgreSQL while both systems run in parallel until the cutover is validated.

1

Assess

Document HSQLDB schema; identify all tables, constraints, data types, and seed data

2

Design

Design PostgreSQL target schema per bounded context; define Flyway migration scripts

3

Export

Export HSQLDB data to CSV using CSVWRITE; capture row counts for validation

4

Transform

Python script: hash plain-text passwords with bcrypt; map column renames; drop SSN

5

Load

Import transformed CSVs into PostgreSQL using \COPY; run validation queries

6

Cutover

Switch services to PostgreSQL; deprecate HSQLDB; run final reconciliation

5. Data Migration Scripts

Step 1 — Export from HSQLDB (Bash)

# Run within HSQLDB console or via JDBC tool
CALL CSVWRITE('/tmp/customers.csv',
  'SELECT ID, FIRSTNAME, LASTNAME, EMAIL, PASSWD, DOB,
          ADDRESS, CITY, STATE, ZIP, PHONE
   FROM CUSTOMER');

CALL CSVWRITE('/tmp/vehicles.csv',
  'SELECT ID, MAKE, MODEL, YEAR, VIN, CUSTOMER_ID
   FROM VEHICLE');

CALL CSVWRITE('/tmp/user_roles.csv',
  'SELECT UR.ID, C.EMAIL, UR.ROLE_NAME
   FROM USER_ROLE UR JOIN CUSTOMER C ON C.EMAIL = UR.EMAIL');

Step 2 — Transform Passwords (Python + bcrypt)

import csv, bcrypt, sys

input_file  = '/tmp/customers.csv'
output_file = '/tmp/customers_hashed.csv'

with open(input_file) as fin, open(output_file, 'w', newline='') as fout:
    reader = csv.DictReader(fin)
    fieldnames = ['id','first_name','last_name','email',
                  'password_hash','date_of_birth','address',
                  'city','state','zip','phone']
    writer = csv.DictWriter(fout, fieldnames=fieldnames)
    writer.writeheader()
    for row in reader:
        hashed = bcrypt.hashpw(
            row['PASSWD'].encode('utf-8'),
            bcrypt.gensalt(rounds=12)
        ).decode('utf-8')
        writer.writerow({
            'id': row['ID'],
            'first_name': row['FIRSTNAME'],
            'last_name': row['LASTNAME'],
            'email': row['EMAIL'],
            'password_hash': hashed,
            'date_of_birth': row['DOB'],
            'address': row['ADDRESS'],
            'city': row['CITY'],
            'state': row['STATE'],
            'zip': row['ZIP'],
            'phone': row['PHONE']
        })
print("Password transformation complete.")

Step 3 — PostgreSQL Import (\COPY)

-- Connect: psql -h localhost -U pksadmin -d pksdb

\COPY users(id,first_name,last_name,email,password_hash,
            date_of_birth,address,city,state,zip,phone)
  FROM '/tmp/customers_hashed.csv'
  WITH (FORMAT CSV, HEADER TRUE);

\COPY vehicles(id,make,model,year,vin,owner_email)
  FROM '/tmp/vehicles_mapped.csv'
  WITH (FORMAT CSV, HEADER TRUE);

\COPY user_roles(user_id,role)
  FROM '/tmp/user_roles_mapped.csv'
  WITH (FORMAT CSV, HEADER TRUE);

-- Reset sequences after bulk load
SELECT setval('users_id_seq', (SELECT MAX(id) FROM users));
SELECT setval('vehicles_id_seq', (SELECT MAX(id) FROM vehicles));

Step 4 — Validation Queries

-- Row count reconciliation
SELECT 'users'    AS tbl, COUNT(*) FROM users
UNION ALL
SELECT 'vehicles' AS tbl, COUNT(*) FROM vehicles
UNION ALL
SELECT 'user_roles' AS tbl, COUNT(*) FROM user_roles;

-- Verify no plain-text passwords survived
SELECT COUNT(*) FROM users
WHERE password_hash NOT LIKE '$2a$%';
-- Expected: 0

-- Verify admin role exists
SELECT u.email, ur.role
FROM users u JOIN user_roles ur ON u.id = ur.user_id
WHERE ur.role = 'ROLE_ADMIN';

7. Rollback Plan

To roll back the schema, restore from a database snapshot taken before the init-db.sh run.

# Take snapshot before migration (Phase 3)
pg_dump -h localhost -U pksadmin -d pksdb \
  --format=custom \
  --file=/backups/pksdb_pre_migration_$(date +%Y%m%d%H%M).dump

# Restore if rollback required
pg_restore -h localhost -U pksadmin -d pksdb_rollback \
  --clean --if-exists \
  /backups/pksdb_pre_migration_20250101_1200.dump

# Re-point services to original HSQLDB (revert application.yml)
# spring.datasource.url: jdbc:hsqldb:file:/data/pksdb;shutdown=true
kubectl set env deployment/user-service \
  DB_URL="jdbc:hsqldb:file:/data/pksdb;shutdown=true" \
  -n pks-insurance
