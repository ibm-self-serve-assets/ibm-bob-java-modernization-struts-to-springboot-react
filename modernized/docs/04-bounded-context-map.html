1. Introduction

A Bounded Context Map visualises how different DDD contexts relate to each other. Key DDD relationship patterns used here: Open Host Service (OHS) — a provider publishes a formal API; Customer/Supplier — downstream is a "customer" of upstream, with influence over priorities; Conformist — downstream must conform to upstream's model without negotiation; Anti-Corruption Layer (ACL) — downstream translates upstream's model to protect its own.
