2. Quick Start via Docker Compose

The fastest way to run the entire platform locally. All services, the database, and the frontend start in one command.

  1. Clone the repository 
    git clone https://github.com/your-org/pks-auto-insurance.git
cd pks-auto-insurance
  2. Build all Maven projects 
    mvn clean package -DskipTests --file user-service/pom.xml
mvn clean package -DskipTests --file vehicle-service/pom.xml
mvn clean package -DskipTests --file premium-service/pom.xml
  3. Start all services with Docker Compose 
    docker compose up -d
# Starts: postgres, user-service, vehicle-service,
#         premium-service, api-gateway, frontend
  4. Verify all containers are healthy 
    docker compose ps
# All services should show STATUS: healthy
# Allow ~30 seconds for Spring Boot startup
  5. Access the application
    🌐 Frontend: http://localhost:3000
    🔌 API Gateway: http://localhost:8080
    👤 user-service health: http://localhost:8081/actuator/health
    🚗 vehicle-service health: http://localhost:8082/actuator/health
    💰 premium-service health: http://localhost:8083/actuator/health

3. Manual Development Setup

3a. Start PostgreSQL (Docker)

docker run -d \
  --name pks-postgres \
  -e POSTGRES_USER=pksadmin \
  -e POSTGRES_PASSWORD=pkspassword \
  -e POSTGRES_DB=pksdb \
  -p 5432:5432 \
  postgres:15-alpine

3b. Initialise Database Schema

# Run once before starting any service. Safe to re-run.
cd modernized
./scripts/init-db.sh

# Override defaults if needed:
DB_HOST=localhost DB_PASSWORD=pkspassword ./scripts/init-db.sh

3c. Run user-service Locally

cd user-service
mvn spring-boot:run \
  -Dspring-boot.run.arguments="--DB_HOST=localhost --DB_PORT=5432 --DB_NAME=pksdb --DB_USER=pksadmin --DB_PASSWORD=pkspassword --JWT_SECRET=local-dev-secret-minimum-32-chars"

3d. Run vehicle-service Locally

cd vehicle-service
mvn spring-boot:run \
  -Dspring-boot.run.arguments="--DB_HOST=localhost --DB_PORT=5432 --DB_NAME=pksdb --DB_USER=pksadmin --DB_PASSWORD=pkspassword --JWT_SECRET=local-dev-secret-minimum-32-chars"

3e. Run premium-service Locally

cd premium-service
mvn spring-boot:run \
  -Dspring-boot.run.arguments="--JWT_SECRET=local-dev-secret-minimum-32-chars --VEHICLE_SERVICE_URL=http://localhost:8082 --USER_SERVICE_URL=http://localhost:8081"

3f. Run React Frontend

cd frontend
npm install
npm run dev
# Frontend starts at http://localhost:3000
# Proxies API calls to http://localhost:8080 (api-gateway)

4. Testing the API Manually

Register a new user

curl -X POST http://localhost:8081/api/users/register \
  -H "Content-Type: application/json" \
  -d '{
    "firstName": "Jane",
    "lastName":  "Doe",
    "email":     "jane.doe@example.com",
    "password":  "SecurePass123!",
    "dateOfBirth": "1990-05-15",
    "address":   "123 Main St",
    "city":      "Chicago",
    "state":     "IL",
    "zip":       "60601",
    "phone":     "312-555-0101"
  }'

Login and capture JWT token

TOKEN=$(curl -s -X POST http://localhost:8081/api/users/login \
  -H "Content-Type: application/json" \
  -d '{"email":"jane.doe@example.com","password":"SecurePass123!"}' \
  | python3 -c "import sys,json; print(json.load(sys.stdin)['token'])")
echo "JWT: $TOKEN"

Register a vehicle

curl -X POST http://localhost:8082/api/vehicles \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"make":"Toyota","model":"Camry","year":2021,"vin":"1HGCM82633A004352","color":"Blue"}'

Get a premium quote

curl -X POST http://localhost:8083/api/premium/quote \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"vehicleId": 1, "userEmail": "jane.doe@example.com"}'

5. Running Tests Locally

Unit Tests (all services)

# Run unit tests for a specific service
mvn test -f user-service/pom.xml

# Run all services' unit tests from monorepo root
for svc in user-service vehicle-service premium-service; do
  echo "=== Testing $svc ==="
  mvn test -f $svc/pom.xml
done

Integration Tests (requires Docker for Testcontainers)

# Testcontainers will automatically pull and start postgres:15-alpine
# Ensure Docker Desktop is running!
mvn verify -f user-service/pom.xml -Pfailsafe
mvn verify -f vehicle-service/pom.xml -Pfailsafe

Frontend Tests

cd frontend
npm test           # Jest unit tests
npm run test:e2e   # Playwright e2e tests (requires running backend)

6. Troubleshooting

ProblemSymptomSolution
Port in useAddress already in use :8081Run lsof -i :8081 and kill the process; or change port in application.yml
PostgreSQL not startingContainer exits immediatelyCheck docker logs pks-postgres; ensure port 5432 is free; remove old volume: docker volume rm pks-postgres-data
Schema already existsrelation "users" already existsinit-db.sh uses IF NOT EXISTS — safe to re-run; or drop tables manually and re-run
JWT errorsSignatureException or 401 on all requestsEnsure all services use the same JWT_SECRET value (≥32 chars)
premium-service can't reach vehicle-serviceConnection refused localhost:8082Ensure vehicle-service is running; verify VEHICLE_SERVICE_URL env var is set correctly
Apple Silicon Docker issuesno matching manifest for linux/arm64Add platform: linux/amd64 in docker-compose.yml for images without ARM builds, or use Rosetta 2: enable in Docker Desktop → Settings → Features
React CORS errorsCORS policy: No 'Access-Control-Allow-Origin'Ensure frontend Vite proxy is configured: vite.config.ts should proxy /api to http://localhost:8080
Maven using wrong Javaerror: source release 21 requires target release 21Set JAVA_HOME=$(/usr/libexec/java_home -v 21) in your ~/.zshrc or ~/.bashrc

8. Useful Development Commands

# --- Docker Compose ---
docker compose up -d                          # Start all in background
docker compose down -v                       # Stop all + remove volumes
docker compose logs -f user-service          # Tail user-service logs
docker compose restart premium-service       # Restart one service

# --- PostgreSQL ---
psql -h localhost -U pksadmin -d pksdb      # Open psql shell
psql -h localhost -U pksadmin -d pksdb \
  -c "SELECT * FROM users LIMIT 5;"          # Quick query
psql -h localhost -U pksadmin -d pksdb \
  -c "\dt"                                    # List all tables

# --- Maven ---
mvn clean package -DskipTests               # Fast build (no tests)
mvn test -Dtest=UserServiceTest             # Run one test class
mvn verify -Pfailsafe                       # Run integration tests
mvn jacoco:report                            # Generate coverage report
# Coverage HTML at: target/site/jacoco/index.html

# --- Frontend ---
npm run build                                # Production build
npm run lint                                 # ESLint check
npm run type-check                           # TypeScript type check

9. Default Login Credentials

⚠️ These credentials are for local development only. Change them before deploying to any shared or production environment.

RoleEmailPasswordHow Created
Admin adminadmin (or admin@pks.com) adminpass Seeded by scripts/sql/01-users.sql via init-db.sh; password stored as BCrypt hash
Regular User Your choice Your choice Create via POST /api/users/register or the frontend /register page

💡 Tip: To reset the admin password locally, connect with psql and run:
UPDATE users SET password_hash = '$2a$12$...' WHERE email = 'admin@pks.com';
Generate a new BCrypt hash at https://bcrypt.online or with the Python bcrypt library.
